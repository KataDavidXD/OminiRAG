#### 核心问题

1. 如何标准化 RAG？
  - 基于 WTB
2. 如何在已经标准化的 RAG 组件空间里，找到高质量的 pipeline 配置？（组件离散组合，例如 embedding、chunking、retriever、reranker、top-k、prompt 或 generator 模式。）
  - Backward greedy + Mutation jump
3. 如何加速？大模型跑一次 benchmark 太慢，组合优化时间成本太大。
  - Sampling (Metropolis-Hastings 或 Hamiltonian Monte Carlo)

#### 架构分层设计

问题：目前的rag工作大多是在rag流程一个节点做一些额外工作 benchmark跑分最高。
我们：优化与标准化框架

- 外层优化： Backward search + Mutation jump，用来在组件组合空间中移动。
- 内层评估： Sampling-based partial benchmark estimation，用来决定一个候选配置是否值得继续花预算。
- 统一底座： WTB infra 负责固定执行环境、缓存、workspace、日志与 checkpoint 语义，使实验结果可重放、可比较、可调试。

优化目标：
$$F(x) = \frac{1}{N} \sum_{i} \ell(x, q_i)$$
其中，$x$ 是一个 RAG 配置，$q_i$ 是 benchmark item，$\ell$ 是质量与成本联合后的单题 reward。真正昂贵的是每次都完整计算 $F(x)$，所以 sampling 的职责是用尽可能少的题估计 $F(x)$。

#### Hamiltonian Monte Carlo (HMC) Sampling

HMC 不能直接在离散题号上运行，因此这里不把 HMC 放在 item-level，而是放在 cluster-weight level。

1. 先把 benchmark 聚成若干簇（如 easy factual、multi-hop、retrieval-hard、rerank-sensitive 等）。
2. 用连续变量 $\theta$ 表示各簇预算权重的 logits，再定义可导的 cluster utility $U_x(\theta)$。
3. HMC 的输出应该是这一轮更应偏向哪些簇。具体题目仍然在簇内再抽样。
